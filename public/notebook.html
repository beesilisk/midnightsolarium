windows management & troubleshooting tools

Windows comes prepackaged with a wide range of tools and features, and a handful of them are actually useful! Yes, really. Lets go over some troubleshooting and system management programs that don’t get deleted by those Windows 10 & 11 debloater scripts.

Task Manager (taskmgr.exe)

If you’re only familiar with one of these tools, you’re familiar with Task Manager. It has a variety of uses, such as managing apps & services, logged-in accounts, and system performance.

Also there are an absurd amount of ways to open Task Manager, including but not limited to:

  • typing “taskmgr” in the Run command
  • right-clicking the taskbar
  • CTRL + ALT + DEL
  • Win + X
  • CTRL + Shift + ESC

Services

A service is a type of system process that doesn’t require user interaction. Your computer will have a list of services running in the background at all times in order to function. The Task Manager Services tab shows you that list of running services as well as their identification information (description, group, status).

Startup

This tab tells you about the services that begin at startup. You can enable or disable the programs, or select startup type to specify how the program launches. It can also show you the impact each program has on boot time.

taskmgr.exe

Performance

The Performance tab shows the resource utilization of the CPU, Memory, Disk I/O, and network. It’s especially useful when the computer is running slowly, since it can help quickly identify bottlenecking.

When it comes to CPU utilization, occasional high spikes in utilization are normal. But if the utilization is consistently high, consider either upgrading other resources or running fewer processes.

Processes

This tab lists all the processes currently running on the system, and outlines each one’s consumption of various resources. By default it only shows the processes of the user that’s currently logged in, but it can also list the processes of all the accounts on the system. This tab can also be used to end tasks.

Users

Shows all the users who are currently logged in, tells you which apps they’re running, and compare the resource utilization of each account. You can also use Task Manager to send messages to users, and sign them out if needed.

resmon.exe

Resource Monitor (resmon.exe)

Resource Monitor is a lot like the Performance and Processes tabs in Task Manager – it lets you live monitor the system’s resource usage, as well as end processes. The main benefit of using Resource Monitor is that it’s easier to isolate a particular process or service to see its resource usage.

System Information (msinfo32.exe)

System Information shows details about the computer’s hardware and software. You would use it if you need to know about the components, peripherals, drivers, services, and network status. The tool doesn’t let you make changes to the system, but it’s useful for the information gathering stage of the troubleshooting process.

System Configuration (msconfig.exe)

The System Configuration utility lets you change the way Windows behaves when it boots. You would use it if you want to change the default OS, enable or disable Safe boot (aka Safe Mode), or set a timer for how long the boot potions will be displayed.

The tool also lets you choose which services will run at startup – but if you want to change which apps run on startup, you’ll want to go to Task Manager.

Registry Editor (regedit.exe)

The registry is a database that stores configuration settings related to the OS, device, and software. You can alter pretty much every aspect of Windows through the Registry Editor. It’s a powerful tool if you know what you’re doing, but it should be approached with caution. Mistakes can cause serious issues, up to and including bricking the entire system.

Disk Cleanup (cleanmgr.exe)

If your storage drive is running low on space, Disk Cleanup can be used to free up room by deleting files. The files it deletes are typically unimportant, like temporary files and the recycle bin, but you can review the files before they’re deleted and specify which types of files you’d like to remove or keep.

Disk Defragment (dfrgui.exe)

Unlike Disk Cleanup, the Disk Defragment tool does not delete anything from the system. Instead, it reorganizes the data blocks spread throughout the physical diskThere are technically ways to defragment an SSD too, but doing so is even less necessary – if fact, it’s more likely to damage the drive due to unnecessary wear and tear so that files are kept in the same area, speeding up the disk’s read times. This tool isn’t as necessary as it used to be now that Windows defragments drives automatically, but using it to optimize performance won’t hurt anything.

MMC (mmc.exe)

The Microsoft Management Console is less of a tool on its own and more of a container used to bring numerous tools into a single space.

MMC snap-ins are all programs that can be accessed on their own, but the MMC is a convenient space for pulling up whichever tools you need at once. It also lots you save the console setup, so you can quickly pull up whichever assortment of tools you find yourself using most.

Event Viewer (eventvwr.msc)

Event Viewer shows Windows’ log files related to events. It’s typically used to review previous errors, and is especially useful when you want to see old error messages.

There are 4 main log files:

  • System – log for events related to the core OS
  • Application – log for 3rd-party software and non-core processes
  • Security – log for system audit data
  • Setup – log for events that occur during installation

It’s worth mentioning that even though Event Viewer is often associated with errors, not every “event” will be critical errors. It also records audit logs, occurrences that OS deems noteworthy, and warnings for potential future issues.

Disk Management (diskmgmt.msc)

This shows you information regarding the system’s storage drives, and lets you create and format drives on physical disks. Most of this tool’s functions and more can be done using the command-line tool diskpart, but Disk Management wraps it in an easy-to-use GUI. Be careful when using this tool, because it’s very easy to make irreversible changes and lose data.

taskschd.msc

Task Scheduler (taskschd.msc)

Task Scheduler lets you automate tasks so that they run at specified times. It’s used to launch programs and scripts at a predetermined time, or when certain criteria is met, such as at login or when the computer is idle. You often see it used for simple tasks like turning computers on or off, sending notifications, or creating backups, but it can also be used to run more advanced scripts.

Device Manager (devmgmt.msc)

Lets you view the status of devices, view their properties, and configure their parameters. It can be used to enable or disable hardware, which can be useful for troubleshooting. But it’s more frequently used to manage drivers, allowing you to enable or disable them, roll back to previous versions of drivers, or identify errors and software conflicts.

Certificate Manager (certmgr.msc)

Certificate Manager is used to see the certificates used by the OS, as well as any web browsers installed on the system. It also lets you install and remove certificates, and allows you to modify its trust lists and revocation lists. When troubleshooting issues related to authentication, it’s good to check the certificates to make sure any software or web applications being used are trustworthy.

Local Users and Groups (lusrmgr.msc)

If you only a Home version of Windows installed, you won’t be able to find this tool. For those of us with Pro or higher, Local Users and Groups provides an in-depth console for managing accounts on the local network. From here you can create, modify and disable accounts, manage their passwords, assign account permissions, or connect them to shared drives. You can also manage the permissions of individual users, or assign permissions to entire groups.

gpedit.msc

Local Group Policy Editor (gpedit.msc)

The Group Policy Editor provides a GUI for configuring Windows settings. It technically performs a similar function to the Registry Editor, but its GUI is a lot more user-friendly, so it’s a shame it’s not available on Home versions of Windows. But I suppose it makes sense – this tool is more useful for modifying broader system settings within an Active Directory domain, as opposed to managing managing local accounts via lusrmgr.msc.

As well as letting you configure OS settings, vendors can also write administrative templates that make third-party software configurable through the Group Policy Editor.

Performance Monitor (perfmon.msc)

I know some confuse Performance Monitor with Resource Monitor, which makes sense. Both tools are used to monitor system resource utilization. But once you actually use these tools, it’s clear that these are two very different pieces of software.

When you first open up Performance Monitor, you won’t be shown any monitors or live feeds. You need to manually select which resource you want it to monitor – and the list of resources it provides is far more exhaustive than most other monitoring tools. Once selected, Performance Monitor will start generating a counter or trace log.

It’s considered best practice to use Performance Monitor to measure resource utilization at different times of the day, or at regular intervals throughout the year. This gives you a thorough understanding of the system’s typical performance, and gives you a frame of reference when troubleshooting specific issues.

introduction to windows client networking

This goes over the basics of network configuration on a Windows computer, and can work as a general guide for connecting your client computer (aka a computer that uses the resources of another computer or server) to a preexisting network. I also provide some brief explanations on basic networking concepts, don’t expect me to go over setting up an entire home network.

Client Network Configuration

Your network adapter should at least be configured with an IP address and a subnet mask. These are the minimum requirements for creating a functional network.

IP Addressing Scheme

If you want your computer to communicate with a network, it needs to receive an IP address. On Windows, there are 3 ways you can configure the IP address:

  • Network & Internet settings
  • the Network Connections applet
  • the Settings app

By default, your computer’s Ethernet or Wi-Fi adapter will come pre-installed with both IPv4 and IPv6 compatibility. The client’s adapter will automatically use whichever version the network uses, so you usually don’t need to worry about configuring the wrong version on the client side. If both IP versions are available, Windows will usually choose IPv6, but you can also manually configure which one the computer uses.

Static vs. Dynamic

IP addresses can be configured to be dynamic or static.

Most clients are configured with dynamic addressing, using the network’s DHCP server (usually built into the router) to change IP addresses periodically as well as each time the device reconnects to the network.

Static IP addresses are usually used for regularly accessed network resources like printers and servers.

If your device doesn’t have a static IP address and can’t connect to a DHCP server, Windows will automatically assign an APIPA address. This lets the computer communicate with the local network, but it won’t be able to connect to the Internet or any other network outside the local subnet.

Subnet Mask

Every computer and network device on the local network needs to have the same subnet mask so that devices within the network can communicate with one another (including the routers and access points that connect you to the Internet). Devices are given a subnet mask by the DHCP server alongside the IP address.

Rather than asking for the subnet mask, Windows might ask you for the subnet prefix length. That is... a whole can of worms that I’m not about to get into. For now, all you need to know is:

  • A subnet prefix is a machine-readable binary representation of your subnet mask
  • Each subnet mask corresponds to a distinct subnet prefix length
  • If you know your subnet mask, you can look up that subnet’s prefix length (or CIDR, which is the same number)
  • Chances are your prefix length is 24 (that’s 255.255.255.0)

Gateway

A gateway address is an address for the router. Using this, your computer is able to forward network traffic from the local network to other networks such as the Internet. To configure the gateway on Windows, all you really need to do is type in your router’s IP address.

DNS

If you’re using the Internet, a DNS server is needed to resolve IP addresses to their domain names. This server might be within your local network, or you can query from a public DNS server. Windows lets you provide the address for two separate DNS servers so that the OS has a second server to fall back one in case the first becomes unavailable.

Different Network Connections

To join your computer to a network, you’ll need to establishing a connection between the computer and the network adapter located on a switch or access point. The exact process for connecting devices will depend on the specific type of connection you’re trying to establish.

Wired Connections

Wired network connections are almost always Ethernet-based. The link between your computer and the adapter/switch will usually be established as soon as the cable is plugged in.

In Windows, each wired adapter is assigned a name. The first one is called Ethernet, the next Ethernet2, and so on.

If you need to configure any Ethernet settings manually, there are 2 ways to go about it:

  • go to Device Manager, locate the adapter, right-click on it and select 'Properties', and select the 'Advanced' tab to change the settings
  • go to Network & Internet settings, and enter the status page to confgure the adapter

Wireless Connections

To establish a wireless connection, go to your notification area and select the network status icon. Assuming you’ve set your access point to broadcast the SSID, your network should appear on a list of available networks.

To connect, click on your network and enter the password. Once connected, you can choose the Connect automatically option so Windows will connect to the network whenever it’s within range.

The adapter is configured using Device Manager. The adapter will need to support an 802.11 standard that’s compatible with your device.

WWAN

A Wireless Wide Area Network (WWAN) adapter is an adapter that connects cellular devices to the Internet using the cell provider’s network. The bandwidth will depend on which one is supported by both the adapter and the cell tower.

If your network’s SSID isn’t being publicly broadcast, you’ll need to input your WWAN settings manually. To do this, go to Network & Internet, select 'Wi-Fi', 'Manage known networks', and then 'Add a new network'.

VPNs

A VPN allows you to connect to another network using an encrypted tunnel.

Windows supports several different types of VPNs. If yours is supported (and it probably is), the connection can be configured via Network & Internet settings.

Once connected, the network connection can be seen on the network status icon. You can right click the icon and select the VPN connection icon to connect, disconnect, or modify the connection.

If your VPN isn’t supported by Windows, that doesn’t mean you can’t use it. You’ll just need to install a third-party client software.

Workgroups & Domains

On Windows, there are 3 main network setups:

  • Domain – Used in larger networks. Managed by a domain controller using Active Directory. New clients must be joined to the domain.
  • Workgroup – Used in smaller networks (max. 20 inbound sessions per client) and SOHO setups. OS acts as a both a server and client.
  • Homegroup – Like a workgroup, but only works on a single private network. Depreciated, but still exists on older versions of Windows 10 and prior.

Shared Resources

Network shares allow files & folders to be shared by some or all users within a network. For client computers, the network share will look similar to a local disk.

The administrator can choose to make a network share invisible to regular users by putting $ at the end of the share name. Other users will still be able to access the shared resource if they know the name, but it won’t be displayed on a list of shared resources.

Some administrative shares are created automatically by Windows. One example is Admin$, which is used for remote administration. These shares are only visible to admin accounts, and most can only be removed by editing the registry.

Mapped Drives

Shared resources can be mapped to a specific drive letter on a storage device. If a user needs to access a shared resource, they one need to navigate to that specific drive. This makes the resources easy for the network group to access; finding a shared resource isn’t very different than finding a local resource.

To map a network drive, open up File Explorer and navigate to 'My PC'. Select 'Map network drive'. Using the dialog box that pops up, you’ll then choose a drive letter, enter the folder path, and check “reconnect at sign-in” if you want the connection to be persistent.

Printers

Printer sharing allows computers to print to printers connected to the local network. You can connect your computer to a shared printer using File Explorer, or the Printers & scanners section of the Settings app.

To share a printer that’s connected to your computer, go to Printers & scanners, select the printer, and choose 'Manage'. Click 'printer properties', and configure the share using the Sharing tab (from here you can make sure the correct drivers are installed, too). You can also go to the Security tab to specify which users are able to use the printer.

Windows Firewall Settings

Your network connection is controlled by the settings imposed by your firewall. Most versions of Windows comes prepackaged with Windows Defender Firewall. You can use a 3rd-party firewall if you prefer, but the one that comes with Windows is good enough for most people, so that’s what we’ll be focusing on.

Configuration

You can configure Windows Defender Firewall either by going to the Windows Defender Security Center and accessing the Firewall & network protection page, or through the Windows Defender Firewall applet within the Control Panel.

From there you can turn the firewall on or off, or choose to block all incoming connections (assuming you have administrative permissions). When every connection is blocked, this will include any exceptions outlined in your exception list. This will prevent everyone on the network from accessing a shared resource, which may sometimes be necessary for security.

Application Restrictions and Exceptions

Windows Firewall can see all the applications running on the host system. You can allow or block the listed programs by going to the Windows Firewall status page, select ‘Allow an app through the firewall’. You can choose whether the software is available on the private network, public network, neither, or both, as well as configure these settings for specific port numbers.

Proxy Settings

A proxy server works as a type of gateway for the network. The client PC will pass Internet requests to the proxy, which then forwards them to the Internet.

Proxy servers are often implemented to improve security. For example, a web proxy is often used to filter access to the internet from a local network. They can also be used to improve performance, such as by caching pages that are being accessed by multiple clients.

To configure the proxy settings on a Windows client, their computer will need to be configured with the IP address and TCP port being used to forward traffic via the proxy. You can do this through Network & Internet settings.

That said, some proxies are auto-configuring, and some don’t need to be configured on the client side at all.

Public vs. Private Networks

A public network is accessible to everyone within the network’s range. Once a network is made public, Windows Firewall will automatically block all access to the host and make it undiscoverable. This makes it harder for people on the public network to gain access to system resources.

Access to a private network is limited to those who provide proper authentication. The firewall is set to allow host discovery and resource sharing, since it’s assumed that anyone on the network is trusted.

File Explorer Network Paths

Once network discovery is enabled via Network & Internet settings, other devices on the network will be accessible through File Explorer’s navigation pane.

Windows uses universal naming convention (UNC) syntax to address network hosts and resources. The syntax for a UNC network path is:

\\Host\Path

“Host” refers to the host name, FQDN, or IP address of the server. “Path” is a shared folder or file path.

Metered Connections and Limitations

It’s common for cell providers to impose data limits (some of which are not made apparent to the user). Because of this, it’s important to make sure there aren’t any apps using up a lot of data. To prevent this, you can make the connection a metered network and set a monthly data limit. You can also modify whether the OS an apps can update automatically, or turn off OneDrive syncing.

some logical security measures

What Do You Mean by "Logical Security"?

Logical security refers to the digital security measures taken to protect computers and the data within them. You can think of it as the equal opposite of physical security, i.e. making sure nobody can just walk up and steal your laptop.

Principle of Least Privilege

This principle states that users should be given only what they absolutely need in order to perform their tasks. Everything else should be restricted.

Usually this term is used in reference to user account privileges. After all, not everyone needs to be an administrator. Granting every user high-level access would increase the risk of accidents and security breaches. By restricting a user's privileges, you also restrict the privileges of any malware that infects the system.

This principle may seem straightforward, but its importance should not be overlooked. Make sure all users are assigned to the correct group, don't give users the ability to modify newly installed programs, and don't start granting higher privileges out of laziness.

ACLs

Access Control Lists outline the permissions and restrictions set for a user or system. They dictate which accounts are allowed to particular files, folders, programs, or network resources.

As well as being built into the OS, ACLs can also be configured on routers, firewalls, and switches for traffic filtering.

MFA

Multifactor authentication verifies the identity of a user by having them use at least 2 different methods of gaining access. Verification methods are split into 3 categories:

  • something you know (e.g. passwords & PINs)
  • something you have (e.g. phones & keyboards)
  • something you are (e.g. figerprints & voice recognition)

Unlike 2-factor authentication, MFA must incorporate different authentication methods from different categories, such as a password + fingerprint combo. MFA can also incorporate more than 2 steps.

Email Filtering

Email is one of the most frequently exploited security vulnerabilities, so it's important to filter unwanted and malicious emails. In organizations, this is usually done through an email gateway. These are able to scan emails to look for malicious attachments and block unwanted emails before they can reach a user's inbox. The email gateway can be implemented by using either an onsite server or cloud service.

Hard Tokens

A hard token is a physical object that's required to gain access to network resources. If a person doesn't have the USB, smart card, or key fob on hand, they cannot gain access. This sounds like an elegantly simple security system, but it falls apart if the hard token is lost or stolen.

Soft Tokens

Also known as a one-time password, a soft token is a form of authentication data that's generated for a single use only. This data usually comes in the form of a temporary PIN, often delivered via email, SMS, or voice call. It's used a lot on 2FA and MFA because of its relative convenience and cost-effectiveness compated to hard tokens, but they are more susceptible to hacking.

Authenticator Apps

Authenticator apps let people access resources without a password. Instead, the service or network is registered with the software, making it inaccessible to anyone without an account on the app. If someone with an account tries to gain access (usually done by scanning a QR code, copying an alphanumeric key, or a biometric scan), the app will send them a soft token.

Unlike soft tokens sent over SMS, these apps work without cell service or internet access. The keys are also easier to encrypt and sync across devices. But if you lose access to your account, app, or device, the service becomes inaccessible.

What Are Threats?

Okay, the word “ threat” might seem overly vague. But when you're talking about cybersecurity, a threat is generally considered to be any action that could exploit vulnerabilities in the system or network. If someone takes advantage of that vulnerability, the threat will result in missing, stolen, or damaged data.

Another term I'm gonna be using is “ threat actor”. This refers to whoever is responsible for the threat. They can be a single person or a large group, come from any background, and act under a wide range of motivations.

Insider Threat

An insider threat can come from anyone who's granted permissions on the system. These threat actors are hard to protect against because the attacker is someone who's expected to be on the network.

Sometimes insider threat actors will spend years within an organization for the purpose of collecting data and finding vulnerabilities. Other times the threat actor could be a normally trustworthy person who was bribed or blackmailed. The threat actor could also be a hired contractor or 3rd party vendor with a secret agenda.

Or sometimes the reasons are less interesting. Got an employee who keeps falling for phishing emails? That's an insider threat too.

The motivations behind insider threats are infinitely varied, but we can put them into 2 categories:

  • malicious (e.g. an employee at Evil Inc. realizes that they're the baddies and decides to leak confidential data to a team of plucky rebels), and
  • non-malicious (e.g. your idiot child tries to download My Sim Aquarium from a sketchy website and infects the family computer with malware (sorry dad))

What I'm saying is that, like all threat actors, an insider threat can can come from anyone with any skillset for any reason.

Zero-Day Attack

This describes any sort of threat that exploits an unpatched vulnerability within a system or piece of software – particularly, vulnerabilities that have been present since launch.

These vulnerabilities exist either because they are undiscovered, or because a patch hasn't been made (maybe a patch is still in the works, maybe the flaw is impossible to fully patch, or maybe the vulnerability is being ignored out of negligence). Sometimes zero-day vulnerabilities go years and years without being fixed or even discovered until a threat actor decides to exploit it.

These threats aren't as common as you might expect. Usually software is heavily researched and tested before launch to make sure there aren't any catastrophic security vulnerabilities. That said, a serious design flaw slipping through the cracks could end in catastrophe.

Footprinting

Footprinting is when the threat actor tries to gather information and discover vulnerabilities. The attacker researches their target using a wide range of methods, including collecting publicly available info, scanning network ports, and using social engineering (aka tricking people) to extract information.

Once the attacker has gathered information on their target, they will analyze their data and strategically plan their attack.

While footprinting can be a threat, it's also done by penetration testers in order assess security.

Spoofing

Spoofing happens when the threat actor pretends to be something trustworthy.

Most people think of spoofing as an attacker pretending to be a trustworthy person or from a trustworthy organization, but spoofing can also be:

  • cloning a valid IP or MAC address
  • using a fake digital certificate
  • stealing authentication tokens (such as web cookies)

Essentially, spoofing is a general term that describes impersonation.

On-Path Attack

This is form of spoofing where the threat actor secretly sits in the middle of a 2-way conversation between two devices. The attacker is able to capture the packets of data being sent, analyze their contents, and send them to their destination without anyone else noticing. This threat is also known as a man-in-the-middle attack.

Sometimes the attacker can even modify the packets before sending them to their destination, or prevent the transmission from reaching its destination at all. But usually on-path attacks are trying to steal password hashes. When trying to steal information, it’s in the attacker’s best interest to do as little tampering as possible.

ARP Poisoning

This is a specific type of on-path attack that exploits a weakness in the Address Resolution Protocol (ARP). This is the protocol that identifies each device’s unique MAC via the device’s known IP address. This allows packets of data to be delivered across a network to the correct machine.

Using ARP spoofing, an attacker can trick a machine into associating a MAC address with the attacker’s device instead of the correct one. This lets the attacker assume another device’s identity and use it for spying or spoofing – all without having to steal a username or password.

Brute-Force Attack

Earlier on I mentioned password hashes. These are the way a password is represented in data. There are many different algorithms used for password hashing, they all encrypt the password into a different string of text.

For example, if the password “ password123” using the SHA-1 algorithm, that password will be stored in a database as “ cbfdac6008f9cab4083784cbd1874f76618d2a97”. If that database is breached, the attacker would find the hash but not the original password.

A brute-force attack is a threat based around decrypting hashes back into their original passwords. The attacker would use a script that converts random character strings into their hashed form. If they can recreate a known hash, that means the attacker has found the actual password.

Dictionary Attack

This is another method of decrypting password hashes. Unlike brute-force attacks that generate random character strings to convert into hashes, dictionary attacks generate passwords based on words and phrases. After all, people are more likely to use words in their passwords than a random keysmash.

Some dictionary attack scripts might generate hashes by going through every possible word, but others will favour words and phrases that are known to be common in passwords, such as‘12345’, ‘iloveyou’, ‘P@55w0rd’, or words related to the organization being targeted.

DoS Attacks

A denial of service attack is any attack that makes a service inaccessible. You could do this by exploiting a design failure that causes software to crash, downloading an overly large file, or physically cutting a computer’s power cable.

That said, when most people talk about DoS attack, they're specifically referring to an attack that involves services being overloaded by an onslaught of spoofed traffic. The coordinated influx of network traffic brings the system down under the weight of too many requests.

It's worth noting that sometimes cutting off a service is not the attacker's end goal. DoS attacks are very disruptive and obvious, which makes them great distractions for subtler threats.

DDoS Attacks

A distributed denial of service attack is designed is much harder to stop than a regular DoS attack, which is why you hear about them more often. These attack from multiple locations in order to bring down their target. Attacking from many different sources at once makes them hard to block.

DDoS attacks come in many different varieties, but pretty much all of them work by overwhelming a server with traffic. This is typically done using botnets, which are devices connected to the internet that are under the control of a 3rd party. These devices are often infected with malware, transforming them into remotely controlled attacking machines.

SQL Injections

When an uninvited 3rd party is able to alter an application’s code, this considered to be an injection. There are many injection techniques made for altering different types of code, but SQL injections are probably the most well-known.

The SQL programming language is what allows the data in databases to be manipulate. An injection works by tampering with the SQL queries as they’re being sent to or from the server. This allows the attacker to covertly steal and even modify data stored on the server. Sometimes a SQL injection can create backdoor to the server itself, which provides an opportunity for further attacks on the backend infrastructure.

Cross-Site Scripting (XSS)

When a user clicks a link to a website or enters the site’s URL, that person’s web browsers will generally trust the scripts running on that site. Threat actors can take advantage of this by using a website’s user submission feature – this includes things forums, comment sections, and guestbooks.

Without proper safeguards, features like these allow threat actors to inject malicious scripts onto a trusted site’s code.

In action, cross-site scripting will often look like a comment with a link to another website. If you click that link, the attacker’s malicious code will activate. This could result in stolen data, cookie hijacking and malware infection, just to name a few examples.

NOTE: This is only a very general overview, but amateur webmasters like myself might wanna read more about XSS and how to prevent it.

since this is effectively a summarization of my notes, i wont be going into too much detail, and i might make some generalizations. at most, you should consider this your own jumping off point and do your own research

Trojan

Trojans hide inside the installer packages for legitimate-looking programs. Once installed, it’s granted the same privileges as the program and executable. A trojan may be able to achieve persistence – meaning it hides inside startup folders or scheduled tasks so it can retrigger whenever the computer restarts, even if the system’s credentials are changed.

Rootkit

Rootkits take advantage of operating system functions in order to elevate their privileges. They like to plant themselves within the operating system, sometimes even fusing with the kernel itself (as well as bootloaders, firmware, hypervisors, memory, or applications). This makes them very hard to identify using your typical troubleshooting software. You’ll need to find a 3rd-party remover software specific to whichever rootkit you’re dealing with.

Virus

The word “virus” a broad term used to describe malware that can self-replicate. The initial infection always requires some form of human interaction (e.g. clicking on a malicious download link), but once executed the virus will be granted the same privileges as the infected process. From there it may be able to infect other files, drives, and even spread to other systems via the network or removable drives.

Spyware

After tricking the user into installing it, spyware will covertly collect data on their victim. Its goal is usually either to steal confidential data, or collect data for advertising. While most remain covert, some may make changes to the browser configuration such as allowing tracking cookies or changing the default search engine. Some might even take screenshots or turn on recording devices.

Ransomware

Ransomware is a malware attack that makes some or all of computer’s data / functions inaccessible as a means of extortion. Cryptocurrency is often demanded as payment to release the system, making the attackers hard to track down after the transaction. These attacks sound scarier than they actually are – assuming you make regular external backups (^ω~)☆

Crypto-Ransomware

This is a form of ransomware that encrypts all or part of a user’s data. The victim won’t be able to access those files until the attacker gives them access to an encryption key. The key is actually kept on the computer from the very start, but it’s also encrypted.

NOTE: I’ve seen some people use this term synonymously with ransomware in general, cryptojacking, or any ransomware that demands cryptocurrency as payment. The definition I gave is the one CompTIA uses, and I find it’s the most commonly used definition in general.

Keyloggers

Keyloggers are a form of spyware that tracks every keystroke and mouse click made on the computer. The info is then sent to the attacker for them analyze. This essentially circumvents whichever encryption methods you’re using. These can be used to steal private data, passwords, or credit card numbers. Keyloggers can be implemented using software, 3rd-party websites, and modified USB adapters.

Boot Sector Virus

These are viruses that infect the boot sector or Master Boot Record (MBR) of a disk drive. Once infecting the boot sector, it will then spread to other disks on the computer. By attaching to the code used to start the operating system, the computer will essentially be reinfected each time it restarts – before the antivirus software has a chance to launch. From an outside perspective, it’ll look like the virus keeps coming back each time the computer starts, even after you think its been removed. Luckily, Secure Boot is good at preventing these viruses.

Cryptojacking

Cryptominers perform cryptographic computations to create cryptocurrency – in other words, they mine crypto. Cryptojackers can infect other computers and use their processing power to mine crypto.

NOTE: while some resources like the CompTIA A+ Student Guide (220-1101 & 220-1102) use the terms “cryptomining” and “cryptojacking” interchangeably, i find most people distinguish between the two. Cryptomining refers broadly to the act of mining crypto, while cryptojacking is the act of using malware to mine crypto using another system’s resources.

(dont get me wrong, i have very little love for cryptominers, but i feel using specificity in language is important especially in the age of misinformation but thats just my opinion)

Worms

Unlike a typical virus, worms don’t need to wait for a human victim to unknowingly activate them. They are self-operating, self-replicating, self-activating programs that can wriggle their way into an operating system by exploiting known security vulnerabilities. They often spread between hosts in a network and slow down system resources.

Fileless Malware

While most malware comes in the form of an executable file, fileless malware disguises itself as a legitimate script or document while residing in the computer’s memory. From there it infects and exploits useful system processes such as scripting environments (especially Powershell). This malware hard to detect using regular antimalware software.

backups

Have you you remembered to back up your data recently? Yes, I’m speaking to you specifically. Go back up your data.

Full Backups

There are multiple types of backups for you to choose based on your needs and the technology you have at your disposal.

A full backup creates a copy of everything on the file system. The exact definition of “everything” might vary depending on who you ask, but it can include the operating system, settings, and files. The copy is gonna be more or less the same size as the original.

A full backup can completely restore the dataset on its own, but the drawback is that it’s time-consuming and eats up storage space. This is probably fine if you’re just backing up your personal files, but anything large-scale is probably gonna want a backup system that’s less resource intensive.

Incremental Backups

An incremental backup actually requires 2 types of backups: first, a full backup of the data is created. After that, an incremental backups will only copy the data that’s changed since the full backup. Each subsequent incremental backup will store the data that’s changed since the previous full or incremental backup. The end result is that you have a full backup as well as several smaller increments.

This is a great backup system if you want to store data quickly and take up the least storage space, which is why it’s used a lot in cloud storage. The one downside is that it takes a while to recover all the data, since the process requires combining several backup in order to rebuild the dataset. It’s probably not the best solution if you want the recovery process to be quick and easy.

Differential Backups

This is a lot like an incremental backup, but with one major difference: the backup containing the changed data doesn’t reset each time. Instead of creating several small increments, every change made from the full backup is copied onto a single differential backup. Over time, the differential backup grows bigger and bigger.

To recover the full dataset, you would only need the last full backup and the most recent differential backup. This is still more complex than creating a single full backup, merging two backups is a lot easier than merging several increments. Differential backups are also a good compromise when it comes to backup speed and storage requirements.

Onsite vs. Offsite

An onsite backup is kept in the same physical location as the original data. This may be more convenient and save on bandwidth, but it also means that every version of the data could be lost in the event of a disaster.

In a work environment, it’s generally considered best practice to keep at least one offsite backup. In larger organizations, backups might be sent to a server in another company branch in another part of the world. If you wanna do this for personal files, the best way to do this would probably be through a cloud service.

Backup Frequency

There isn’t any golden rule dictating how often backups should be made. For my own personal files, I have everything automatically backed up at the end of each week. This is probably the most infrequent you’ll ever see data backed up in a work environment. Most businesses prefer to keep daily backups, though depending on their priorities you might also have backups made every few hours or minutes.

The amount of time you keep each backup around also varies. Depending on the data, there might be regulations dictating how long the data should be stored, so it’s best to be aware of company guidelines.

You should regularly test your backup system to make sure everything is being stored properly. The industry standard is to test the backup system once a month at minimum. Highly sensitive data should be checked even more often.

3-2-1 Rule

This is a best practice guideline for businesses. The rule states that 3 copies of a backup should be taken: 2 copies onsite, and 1 kept offsite. This you have 2 versions of the dataset at your immediate disposal, while also keeping it preserved elsewhere.

Grandfather-Father-Son

Also called a “first-in, first-out” rotation schedule, this is a best practice guideline where new backup is made every day. At the end of the week, the most recent daily backup becomes the new weekly backup. The last weekly backup of the month becomes the new monthly backup.

The end result is that you have 3 backups at all times: a daily backup, a weekly backup, and a monthly backup. As time goes on, the most recent of each gets promoted into the next category.

To be clear, this isn’t mutually exclusive to the 3-2-1 backup rule, though you might see businesses using one backup scheme being used but not the other, or they could have their own system that suits their specific needs. But these are considered best practice for a reason: they keep backups available and well-archived.
